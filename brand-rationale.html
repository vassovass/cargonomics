Typography

The prototype pairs two Google Fonts. Both are free, production-ready, and can be self-hosted in WordPress for performance.

Headings — Source Serif 4
Break into shipping & logistics — trained by the people who built the industry.
Variable-weight serif from Adobe. Authoritative and editorial, with optical sizing that renders crisply at both large display sizes and smaller subheadings. I chose it because it signals human curation without feeling stuffy.
Body — Outfit
A free, intensive five-day bootcamp that takes you from zero industry knowledge to placement-ready. No experience needed. No tuition. Just commitment.
Geometric sans-serif with personality. Clean readability at all sizes. I went with Outfit over more common choices (Inter, Poppins, Roboto) to avoid the "template look."

03

How to Change It — CSS Custom Properties

The entire visual identity is controlled by a single block of CSS variables at the top of the stylesheet. Changing any colour or font updates every element that references it. Here is the actual block from the prototype:

:root {
  /* Colours */
  --color-primary: #1E293B;
  --color-secondary: #B87333;
  --color-accent: #115E59;
  --color-ivory: #FAF9F6;
  --color-text: #334155;
  --color-stone: #F0EDE8;
  --color-signal: #DC2626;
  --color-white: #FFFFFF;

  /* Typography */
  --font-heading: 'Source Serif 4', serif;
  --font-body: 'Outfit', sans-serif;

  /* Layout */
  --max-width: 1200px;
  --container-padding: 20px;

  /* Animation */
  --transition-base: 0.3s ease;
  --transition-fast: 0.2s ease;
}

This architecture is deliberate. It means:

  • Rebranding is trivial — change 6 colour values and 2 font references, and the entire site updates. No hunting through CSS for hardcoded hex values.
  • It maps directly to Elementor Global Colours — in WordPress/Elementor, you set up a matching set of Global Colours and Global Fonts. Every widget that references "Primary" or "Secondary" updates globally, the same way the CSS variables work.
  • Design consistency is enforced — developers and content editors can only use colours from the defined palette, preventing drift.
  • Dark mode or theme variants become a matter of overriding the :root block, not rebuilding the site.
Elementor mapping: Elementor Pro's Global Colours panel supports naming (e.g., "Primary," "Copper Accent," "Stone"). Set these up once during the WordPress build, and every widget colour picker will show the brand palette first. If you later change "Copper" from #B87333 to a different value, every button, badge, and accent on the site updates in one save.

Lead Capture Touchpoints

The prototype includes multiple points where a visitor can convert, placed at different stages of engagement. In production, each CTA points to an application form that feeds the CRM:

Touchpoint Type & Placement
Nav CTA "Apply Now" in the sticky navigation bar. Visible at every scroll position. Uses .nav__cta class for distinct analytics tracking.
Hero CTAs Two buttons: "Explore Programs" (soft scroll to programs section) and "Apply Now" (hard conversion). Side by side.
WYDLS card "Apply Now — It's Free" button on the featured program card. Contextual — appears after the visitor has read about the bootcamp.
Journey CTA "Start Your Application" after the 4-step How It Works section. Large, centered, hard to miss.
Desktop modal "Reserve My Free Spot" — appears after 8 seconds for non-converting desktop visitors. First-person CTA phrasing.
Mobile sticky "Apply Free" bar at the bottom of the viewport. Scroll-triggered, dismissible. Avoids Google's interstitial penalty.
Chat buttons Zalo (primary) + WhatsApp (secondary) floating buttons. For prospects who prefer messaging over forms.
Zalo-first: The floating button order puts Zalo on top because it has 81-85% market penetration in Vietnam vs WhatsApp at 6-7%. WhatsApp is there for international contacts. This is the kind of local-market detail that matters.

SEO & Performance Notes

The prototype is built with production-grade markup from the start, not as an afterthought:

  • Semantic HTML5 — <header>, <nav>, <main>, <section>, <footer> (not div soup)
  • Single H1 with logical H2/H3 hierarchy throughout
  • Schema.org structured data — EducationalOrganization + Course (JSON-LD) for rich results
  • Open Graph + Twitter Card meta tags for social sharing
  • Lazy-loaded images below the fold, eager-loaded hero
  • Minimal JavaScript — IntersectionObserver for scroll effects, no heavy frameworks
  • Mobile-first CSS — base styles for small screens, media queries progressively enhance for tablets and desktops
WordPress performance target: Lighthouse 90+ is achievable with WP Rocket (Remove Unused CSS, Delay JS), locally self-hosted fonts via OMGF or manual @font-face, and WebP image conversion via ShortPixel or Imagify. The prototype's minimal JS footprint and semantic structure give the production build a clean starting point.
Target keywords (student recruitment)
  • shipping logistics training vietnam
  • logistics career ho chi minh city
  • free logistics bootcamp
  • shipping industry course vietnam

These are naturally incorporated into the heading hierarchy and body copy without keyword stuffing. The H1, H2s, and meta description all target these terms.

08

Analytics Readiness

The prototype is structured for clean analytics instrumentation in production:

  • Section IDs (#programs, #about, #trainers, #contact) — enable scroll-depth tracking and section engagement measurement
  • Distinct CTA classes (.nav__cta, .btn--copper, .program-card__cta, .modal__cta) — allow event tracking by position and context
  • Modal/sticky bar interactions — show, dismiss, click-through are trackable conversion micro-events
  • Zalo/WhatsApp button clicks — measurable as contact intent events

In production, Google Analytics 4 (or PostHog) would track page views, scroll depth, CTA clicks by section, modal engagement rate, floating button clicks, and application form submissions. These feed into the CRM for full funnel visibility.
