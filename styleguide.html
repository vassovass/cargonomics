Cargonomics Internal Styleguide

Live render of every reusable component on the site. Each entry lists its BEM class, tokens used, data-slot/data-elementor-widget attributes (WordPress migration targets), and a copy-pasteable HTML snippet. Uses the same `style-v7.css`, `tokens-cargonomics.css`, and `v7-fixes.css` as production, so a regression here implies a regression on the live site.

Internal only. This page is noindex, nofollow and is not linked from public nav, site sitemap, or the footer. Reach it via direct URL: /styleguide or /styleguide.html. Complementary to docs/archived-components.md, which holds removed components; this page holds current components. Deprecated components are kept in-page with a .sg-deprecated-badge at the top of their section until they migrate to docs/archived-components.md once the replacement is stable. The trainer card v2 (below) is the first example of this pattern. Each component's "Technical details" panel (click to expand) carries a Used on row naming the live pages where it renders, and must be updated in the same commit whenever a component is added, removed, or moved on a live page. Full rule in .claude/rules/website-conventions.md "Styleguide Usage Log".

Trusted Partners marquee

Technical details

Social-validation band. v3 light treatment (2026-04-16 evening): ivory background, navy typography at 55% opacity by default, full-opacity on hover. Each partner rendered as a typographically-distinct fake wordmark so it reads as a real logo. When real logos arrive, the <span class="trusted-partners__logo"> is swapped for <img class="trusted-partners__logo-img">; the grayscale-default + colour-on-hover pattern is the industry-standard (Bain / Stripe / Figma / ITS Logistics) and accommodates any real partner brand colour. Earlier grid version and v2 dark-navy treatment both archived in docs/archived-components.md.

Trainer card v3, C: Split

Technical details

Two-panel card. Portrait on the left (45%), info panel on the right (55%). Default info shows name, role, one-liner. On hover (desktop) or :focus-within (keyboard, mobile tap), the panel swaps to a three-bullet "what I'll teach you" list. No JS needed. tabindex="0" on the card container plus a visible focus outline handle keyboard and mobile interaction. Stacks vertically on screens under 768px.

Used on:
No live pages (styleguide-only). Swap-ready: change .trainer-card-v3--editorial to .trainer-card-v3--split on index.html #trainers and course.html #course-instructors.
Status:
Styleguide-only. Not live. Available via single-class swap on index.html / course.html.
Class:
.trainer-card-v3, .trainer-card-v3--split, .trainer-card-v3__media, .trainer-card-v3__photo, .trainer-card-v3__info, .trainer-card-v3__face, .trainer-card-v3__face--default, .trainer-card-v3__face--reveal
Interaction:
Pure CSS. :hover and :focus-within swap opacity on the two faces. Card carries tabindex="0" to be keyboard-focusable. Focus outline uses --color-secondary.
Accessibility note:
Hover-on-touch is an anti-pattern in general, so :focus-within is the primary trigger and hover is additive. First tap focuses and reveals, tap outside blurs and hides. Do not replace this with JS unless the pure-CSS behaviour breaks in a target browser.
Tokens:
--color-primary, --color-secondary (focus outline, bullet markers), --color-white, --transition-base.
Elementor target:
Team Member widget with a Repeater for the reveal bullets. ACF fields: portrait, name, role, tagline (one-liner), teaches (Repeater with 3 Text fields).
